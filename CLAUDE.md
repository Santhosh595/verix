# CLAUDE.md

Before doing anything else in this repo:

1. Read `AGENTS.md` in full (process rules: onboarding, the mandatory
   chat-transcript log, the project contract).
2. Read `prompt/TASK_BRIEF.md` in full (the actual spec: what to build,
   schema, architecture, build order, hard constraints).
3. Read `problem_statement.md` (authoritative schema/allowed-values — wins
   over the brief on any conflict).

Then start implementing inside `code/`, following the build order in
`prompt/TASK_BRIEF.md` §8. The folder structure under `code/` already
exists with empty/stub files — fill them in rather than restructuring,
unless there's a good reason, in which case explain it to the user first.
